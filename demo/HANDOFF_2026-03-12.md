# Demo Handoff (2026-03-12)

This handoff summarizes how to run the demo and OpenClaw tests from this repo, with no machine-specific identifiers.

## 1) Repo-Local Paths

- Demo startup script: `demo/vllm/start_remote_demo.sh`
- Demo stop script: `demo/vllm/stop_remote_demo.sh`
- Gateway app: `demo/vllm/server.py`
- Demo UI: `demo/vllm/static/index.html`, `demo/vllm/static/app.js`
- OpenClaw test guide: `demo/openclaw-demo/TESTING_GUIDE.md`
- Sample docs: `demo/openclaw-demo/sample-docs/`

## 2) Start / Stop Demo

Run from repo root:

```bash
GPU_MEMORY_UTILIZATION=0.95 MAX_MODEL_LEN=32768 \
bash demo/vllm/start_remote_demo.sh
```

Stop:

```bash
bash demo/vllm/stop_remote_demo.sh
```

Health check:

```bash
curl -s http://127.0.0.1:8010/healthz
```

## 3) OpenClaw Testing

### Provider / agent expectations

In OpenClaw config, use:

- `baseUrl`: `http://127.0.0.1:8010/v1`
- model `id`: must match backend model id from `GET /v1/models`
- `contextWindow`: `32768` (for OpenClaw context guard)
- `maxTokens`: `4096`
- agent workspace: `demo/openclaw-demo/sample-docs/`

For full OpenClaw JSON examples and step-by-step setup details, refer to:

- `demo/openclaw-demo/TESTING_GUIDE.md` (Section 2: Configure OpenClaw)

### Reset session before each run

```bash
rm -rf ~/.openclaw/agents/docmind/sessions/*
```

### Run

```bash
openclaw agent --agent docmind -m "<your prompt>"
```

## 4) Critical Runtime Parameters

Current startup defaults in `demo/vllm/start_remote_demo.sh`:

- `MAX_MODEL_LEN=32768` (declared max length for compatibility checks)
- `VLLM_RELAXED_KV_CHECK=1` (bypasses startup rejection for oversized declared max length)
- `TRIATTN_RUNTIME_PROTECT_PREFILL=false` (prefill protection disabled)
- `KV_BUDGET=14336` (TriAttention budget)
- `GPU_MEMORY_UTILIZATION=0.95`

Important: declared 32K does not mean real single-GPU KV capacity is 32K. Keep generation budget controlled.

## 5) Gateway Safety Behaviors

Gateway-side protections are enabled:

- clamps oversized completion length (`DEMO_MAX_TOKENS_CAP`, default 4000)
- terminates stalled secondary stream after idle timeout (`DEMO_SECONDARY_STREAM_IDLE_TIMEOUT_S`, default 12s)
- frontend displays KV-cache-insufficient termination message if a backend stalls/preempts

## 6) Logs

- Baseline backend: `demo/vllm/logs/baseline.log`

grep -a -n "Application startup complete\|KV cache check relaxed" demo/vllm/logs/baseline.log
