# Offline Trace Analysis Toolkit

This folder holds helper scripts for working with the large `.pkl` artifacts generated by offline DeepThink runs. The tools avoid modifying the upstream project and keep all experimental work under `development/`.

## Layout
- `convert_offline_pickles.py`: converts an offline result pickle into a lightweight folder containing `metadata.json`, a compressed token/logprob array (`trace_sequences.npz`), and per-trace JSON dumps.
- `run_convert.py`: CLI wrapper around `convert_offline_pickles.py` for one-off conversions. Usage:
  ```bash
  conda run -n dc python development/offline_analysis/run_convert.py \
    outputs/deepseek_r1_qwen3_8b/offline/deepthink_offline_qid4_rid0_20251003_003315.pkl
  ```
- `run_convert_single.py`: simple script variant that hard-codes a target path; useful when experimenting inside the `development` directory.
- `plot_logprob_stats.py`: reads a converted run and draws per-trace log-probability curves. Each trace is resampled to the requested number of bins (default 100) and plotted in blue (correct answer) or red (incorrect). Example:
  ```bash
  conda run -n dc python development/offline_analysis/plot_logprob_stats.py \
    development/offline_parsed/deepthink_offline_qid4_rid0_20251003_003315 --bins 100
  ```
  The script writes `*_logprobs.png` images under `logprob_stats/` inside the run folder.
- `check_accuracy.py`: quick accuracy summary for a raw `.pkl`. Prints total trace count and how many extracted answers match the ground truth.
- `inspect_pickle.py`: dumps the top-level keys and sample fields from a result pickle (requires the `dc` conda environment because it depends on `vllm`).
- `inspect_trace_logprobs.py`: spot-checks the structure of per-token logprob objects inside a pickle (useful before conversion).
- `pkl_read_benchmark.py` / `read_random_pkl.sh`: quick scripts for timing raw pickle reads or sampling a random pickle.

## Typical Workflow
1. **Inspect accuracy** (optional):
   ```bash
   conda run -n dc python development/offline_analysis/check_accuracy.py <path-to-pkl>
   ```
2. **Convert** the pickle to the lightweight layout:
   ```bash
   conda run -n dc python development/offline_analysis/run_convert.py <path-to-pkl> --overwrite
   ```
3. **Generate plots** once conversion finishes:
   ```bash
   conda run -n dc python development/offline_analysis/plot_logprob_stats.py \
     development/offline_parsed/<converted-folder> --bins 100
   ```
4. (Optional) use other helpers for deeper inspection.

All scripts assume the `dc` conda environment is active (or invoked via `conda run -n dc`) so that dependencies like `vllm` and `matplotlib` are available.
