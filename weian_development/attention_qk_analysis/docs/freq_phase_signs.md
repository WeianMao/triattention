# 频段角度与距离的符号约定

为避免在解读频段可视化时产生歧义，这里明确罗列各个步骤的正负号约定。

## Token 与距离 Δ
- 序列位置使用自然索引 `0, 1, 2, …`。
- 在自回归场景下，仅考虑 `i ≥ j` 的 query/key 组合：
  - `i` 表示当前的 **query** token（较新的 token）。
  - `j` 表示历史的 **key** token（较旧的 token）。
- 两者的距离定义为 `Δ = i - j ≥ 0`。在可视化图中，横轴即为这个 Δ。
  - 将当前位置视为 0 时，历史 key 可理解为 `-Δ`，但实现上直接用 Δ 的非负值。

## RoPE 旋转方向
- RoPE 会让较新的 token 额外旋转一个正向角度 `ω · position`。
- 当模型未学习额外相位时，query 与历史 key 的理论相位差正好是 `ω · Δ`，带入重构曲线便是 `cos(ω Δ)`。

## 夹角 φ_f 的计算
1. 对每个频段 `(x, y)` 对恢复 RoPE 之前的向量，得到 `q_pair` 和 `k_pair`（在代码中会先除以 Yarn 的 `attention_scaling`，再做逆旋转）。
2. 分别单位化：`q_unit = q_pair / ||q_pair||`、`k_unit = k_pair / ||k_pair||`。
3. 计算两个已归一化向量的点积与叉积：
   - `dot = q_unit · k_unit`
   - `cross = q_unit_x * k_unit_y − q_unit_y * k_unit_x`
4. 使用带符号的反正切 `atan2(cross, dot)` 得到夹角。
   - 如果 query 相较 key 额外向正方向旋转，则 `cross > 0`、夹角 φ_f > 0。
   - 如果 query 相较 key 反向旋回，则 φ_f < 0。
5. 仅保留下三角 (`i ≥ j`) 的角度值，按所有 `(i, j)` 对求平均/方差，得到 φ_f 与 σ²_f。

## 重构曲线的符号
- **未考虑相位偏移**：使用 Σ_f |Q||K| cos(ω_f Δ)。
- **考虑平均相位偏移**：使用 Σ_f |Q||K| cos(ω_f Δ + φ_f)。
  - φ_f 为上一步得到的平均夹角；与 Δ 的正方向一致，不需要再翻转符号。

综上：Δ 始终取非负值表示“query 相对于历史 key 的距离”；φ_f 直接表征 query 相对 key 额外旋转的方向与大小，并按照 “从 key 旋到 query” 的约定存储在 
[-π, π]。可视化时把这两者直接带入上述公式即可。
