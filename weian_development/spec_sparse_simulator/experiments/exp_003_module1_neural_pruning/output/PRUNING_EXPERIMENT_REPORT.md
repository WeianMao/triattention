# Module 1 神经网络剪枝实验报告

## 摘要

本报告记录了在保持命中率 >= 99.5% 的前提下，简化 Module 1 Key Pruning Network 架构的实验。目标是在保持剪枝效果（低 keys/query）的同时最小化参数量。

**核心发现：** 将 von Mises 核数量从 3 减少到 1（`num_kernels=1`）取得最佳效果：
- **参数量减少 60%**（16,580 vs 基线 41,156）
- **保持 99.50% 命中率**
- **keys/query 略有改善**（71.84 vs 基线 72.48）

## 实验设置

### 基线配置
- `num_kernels`: 3（每个频率带的 von Mises 核数量）
- `mlp_hidden_dim`: 64
- `use_mlp`: True
- **参数量**: 41,156
- **命中率**: 99.51%
- **Keys/Query**: 72.48

### 测试策略
1. **核数量缩减**: 减少 von Mises 核（3 → 1）
2. **MLP 缩减**: 减少 MLP 隐藏层维度（64 → 32/16/8）
3. **移除 MLP**: 用平均池化替代 MLP
4. **组合策略**: 同时缩减核数量和 MLP

## 详细分析

### 策略 1：核数量缩减（推荐）

将 `num_kernels` 从 3 减少到 1：
- **参数量**: 41,156 → 16,580（-60%）
- **命中率**: 保持在 99.50%
- **Keys/Query**: 72.48 → 71.84（略有改善！）
- **阈值**: 0.62（低于基线的 0.99，表明区分能力更强）

**原理分析**: 单个 von Mises 核在适当初始化（kappa=2.5）的情况下，足以捕获关键的频率模式。多个核实际上是冗余的。

### 策略 2：MLP 维度缩减（不推荐）

减少 MLP 隐藏层维度会降低剪枝效果：

| MLP 隐藏层 | 参数量 | Keys/Query | 相对基线变化 |
|------------|--------|------------|--------------|
| 64（基线） | 41,156 | 72.48 | - |
| 32 | 39,044 | 72.57 | +0.1% |
| 16 | 37,988 | 80.55 | +11.1% |
| 8 | 37,460 | 86.32 | +19.1% |

MLP 层对于有效组合核编码至关重要。较小的 MLP 需要保留更多 keys 才能达到相同的命中率。

### 策略 3：平均池化（失败）

用平均池化替代 MLP 完全失败：
- 无法学习有效的剪枝策略（阈值=0.999）
- 保留了 8785 个 keys（基本没有剪枝效果）
- 对核输出进行学习式组合是必不可少的

### 策略 4：组合策略（权衡取舍）

同时缩减核数量和 MLP：

| 配置 | 参数量 | Keys/Query |
|------|--------|------------|
| k=1, h=64 | 16,580 | 71.84 |
| k=1, h=32 | 14,468 | 81.08 |
| k=1, h=16 | 13,412 | 101.37 |

在核数量缩减的基础上进一步缩减 MLP 会导致 keys/query 显著增加。k=1, h=64 配置是最佳平衡点。

## 建议

### 生产环境推荐配置
使用 `num_kernels=1` 配合 `mlp_hidden_dim=64`：
```python
Module1KeyPruningNetwork(
    num_kernels=1,
    mlp_hidden=64,
    kappa_init=2.5  # 单核情况下的重要初始化参数
)
```

**优势：**
- 参数量减少 60%
- 剪枝效果相同或略有提升
- 架构复杂度不增加

### 如需进一步缩减参数
使用 `num_kernels=1` 配合 `mlp_hidden_dim=32`（14,468 参数，-65%）：
- 接受约 12% 的 keys/query 增加（81 vs 72）
- 仍然保持 99.5% 命中率

### 禁忌事项
- 不要使用平均池化（完全无效）
- 不要将 MLP 隐藏层维度降到 32 以下（keys/query 显著恶化）
- 不要在不缩减核数量的情况下单独缩减 MLP（收益甚微）

## 结论

1. **核数量缩减非常有效**: 单个 von Mises 核足以捕获必要信息
2. **MLP 层不可或缺**: 平均池化失败；MLP 隐藏层维度影响剪枝质量
3. **最佳配置**: `num_kernels=1, mlp_hidden=64` 实现参数/性能的最优权衡
4. **边际递减效应**: 超过 k=1 的进一步缩减会以剪枝质量换取微小的参数节省
